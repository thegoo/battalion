# Skills

Skills are reusable guidance for humans or executors that perform work from a Battalion plan.

Battalion may provide skill guidance, but it does not own implementation strategy. Skills must preserve the boundary:

- Battalion owns the WHAT.
- Executors own the HOW.
- Humans make engineering decisions.

No skill runtime is introduced by this repository realignment.

This directory is architectural intent, not a current runtime skill path or package boundary.
